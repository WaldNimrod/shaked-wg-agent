## 1. Problem Statement

The HTML report (`publisher/html_report.py`) contains ~50+ hardcoded German strings for UI labels, status messages, column headers, and tooltips. The embedded JavaScript includes Basel-specific flatfox bbox coordinates (line 297). The entire report assumes a German-speaking Swiss audience.

For Israeli users, the report needs Hebrew labels, RTL layout support, NIS currency display, and Israel-specific map coordinates. This is the largest single i18n effort in the codebase.

## 2. Proposed Solution (concept only)

Implement locale-aware string rendering in the HTML report. Extract all German strings into a locale dictionary. Add RTL CSS support for Hebrew. Replace hardcoded map coordinates with city-definition-driven values. Design requires LOD300 for i18n architecture decisions (template engine, string extraction method, RTL approach).

## 3. Scope

### 3.1 In scope

- Extract ~50+ German UI strings from html_report.py into locale dictionary
- Add Hebrew translations for all extracted strings
- **RTL implementation per AOS Module 11 (RTL_BIDI_STANDARD_v1.0.0.md):**
  - `<html lang="he" dir="rtl">` when IL locale active
  - `bootstrap.rtl.min.css` (Bootstrap 5 RTL variant)
  - CSS logical properties mandatory (`margin-inline-start`, not `margin-left`)
  - `dir="ltr"` wrapper for price tokens (₪ stays left of number)
  - `<bdi>` for unknown-direction inline content
  - `Intl.NumberFormat('he-IL')` for currency formatting
- Replace hardcoded flatfox Basel bbox with city-definition-driven map bounds
- Replace flatfox-specific verification badges with source-agnostic design
- Locale selection based on city definition's `country` field
- **100% locale rule:** IL profile → entire report Hebrew + RTL. CH profile → entire report German + LTR. No mixing.
- LOD300 required: i18n architecture (template engine? gettext? simple dict?)
- Consume `Locale.direction` and `Locale.html_lang` from WP003 locale registry

### 3.2 Out of scope (explicit)

- Full translation management system
- Other languages beyond German and Hebrew
- CLI i18n (covered in S005-P004-WP003)
- Email/notification i18n (covered in S005-P004-WP003)
