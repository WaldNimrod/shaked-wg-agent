## 1. Problem Statement

Once the Yad2 POC (S005-P001-WP001) confirms feasibility and the connector architecture (S005-P001-WP002) is designed, the actual Yad2 scraper must be implemented. This WP covers the production-quality implementation of the Yad2 connector: Hebrew text parsing, NIS price handling, Pardes Hanna region search parameters, and integration with the existing scraping pipeline via the normalizer gateway pattern.

## 2. Proposed Solution (concept only)

Implement a Yad2 connector in `connectors/yad2/` following the architecture defined in S005-P001-WP002. The connector fetches listings from Yad2 using the method determined by the POC, normalizes them to `ScrapedListing` format, and registers via the dynamic scraper registry (S005-P004-WP002).

## 3. Scope

### 3.1 In scope

- Implement Yad2 connector using scraping method from POC findings
- Hebrew text parsing and encoding (UTF-8, RTL markers)
- NIS price extraction and normalization
- Pardes Hanna region search parameters (settlement list from S005-P002-WP001)
- Field mapping to `ScrapedListing` (using generic `price` + `currency="NIS"` from S005-P004-WP001)
- Rate limit compliance (delays, backoff) per POC findings
- Registration in `sources.json` via `connector_class` field
- Error handling for Cloudflare challenges and blocks
- Unit tests for normalizer logic

### 3.2 Out of scope (explicit)

- Scoring model changes for Israeli market
