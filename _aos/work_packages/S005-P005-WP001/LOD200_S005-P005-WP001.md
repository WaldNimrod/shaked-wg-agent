# wgzimmer.ch reCAPTCHA v3 Bypass — Patchright + Persistent Profile — LOD200 Concept

**work_package_id:** S005-P005-WP001
**domain:** shaked-wg-agent
**track:** A — pattern-following scraper modification, single team
**priority:** HIGH — restores a major data source (0 results currently)
**approved_by:** _draft_v1.1.0_
**approved_at:** _awaiting_L-GATE_S_round_2_

## 1. Problem Statement

The wgzimmer.ch scraper (`wgzimmer_pw.py`) currently returns **0 listings** on the production server. Investigation revealed the blocking mechanism is **Google reCAPTCHA v3** (not Cloudflare as previously assumed). The site is served by nginx; the only protection is an invisible reCAPTCHA that scores browser sessions. Headless Playwright Chromium receives a low score (0.1-0.3) from Google's scoring system, causing the server to reject the search request with the message "Das Verarbeiten der Anfrage wurde von Google reCaptcha gestoppt".

wgzimmer.ch is the primary source of WG (shared apartment) listings for Basel — losing it means ~20-40 WG-specific listings are missing from every scan. The flatfox scraper (63 listings) partially compensates but does not cover WG-specific content that wgzimmer provides (roommate descriptions, vegan signals, shared living details).

**Root cause:** reCAPTCHA v3 scores headless browsers without Google cookies as bots. The existing scraper uses standard Playwright which exposes automation markers (`navigator.webdriver`, CDP headers, etc.) that further reduce the reCAPTCHA score.

## 2. Proposed Solution (concept only)

Replace the standard Playwright dependency with **Patchright** (a drop-in Playwright fork that removes automation detection markers) and switch from ephemeral browser contexts to a **persistent browser profile** that retains Google cookies across sessions.

The approach has two phases:
1. **Code change:** Replace `playwright.sync_api` import with `patchright.sync_api` and change `browser.new_context()` to `chromium.launch_persistent_context(user_data_dir=...)`. This is a minimal, non-breaking change to the existing scraper.
2. **One-time setup:** Run the browser once in non-headless mode on the server, manually visit wgzimmer.ch and perform a search. This seeds Google cookies (NID, CONSENT, etc.) in the persistent profile, establishing browser "trust" that dramatically improves the reCAPTCHA v3 score for subsequent headless runs.

**Fallback:** If Patchright + persistent profile still fails, implement a secondary approach using `curl_cffi` with reCAPTCHA tokens from 2Captcha API (~$0.003 per solve, negligible cost at our scale).

## 3. Scope

### 3.1 In scope

**3.1.1 Replace Playwright with Patchright in wgzimmer scraper**

- Add `patchright` to project dependencies (pyproject.toml / requirements)
- In `wgzimmer_pw.py`: change import from `playwright.sync_api` to `patchright.sync_api`
- The Patchright API is identical to Playwright — no other code changes needed for this step
- Maintain the `try/except ImportError` pattern for graceful fallback

**3.1.2 Switch to persistent browser context**

- Replace `pw.chromium.launch(headless=True)` + `browser.new_context(...)` with `pw.chromium.launch_persistent_context(user_data_dir=PROFILE_PATH, headless=True, ...)`
- The `PROFILE_PATH` should be configurable via environment variable or config, defaulting to `~/.shaked-wg/browser-profile/wgzimmer`
- The persistent context retains cookies, localStorage, and browser state between runs
- Must handle profile directory creation if it does not exist
- Must handle profile lock conflicts (if two scans run simultaneously)

**3.1.3 Add human-like interaction delays**

- Before and after form submission, add randomized delays (1-3 seconds) using `page.wait_for_timeout(random.randint(1000, 3000))`
- Add a small random mouse movement before critical interactions (optional, low priority)
- These delays improve reCAPTCHA v3 behavioral scoring

**3.1.4 Browser profile seeding script/instructions**

- Provide a one-time setup script or documented procedure to seed the browser profile:
  1. Run browser with `headless=False` on the server
  2. Navigate to wgzimmer.ch, perform a manual search
  3. Close browser — cookies are now persisted in the profile directory
- This must be done once per server deployment (or when Google cookies expire, typically 6-12 months)
