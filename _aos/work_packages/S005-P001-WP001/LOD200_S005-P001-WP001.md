## 1. Problem Statement

Yad2.co.il is the dominant Israeli rental listing platform (~50K active rental listings). It is protected by aggressive Cloudflare anti-bot measures including JavaScript challenges, CAPTCHA, browser fingerprinting, and rate limiting. The entire Israel expansion architecture (S005-P001-WP002: Yad2 connector) depends on knowing what is technically feasible. Without this POC, no architecture decisions can be made.

This is the single biggest unknown in the Israel expansion. It must be resolved immediately.

## 2. Proposed Solution (concept only)

Conduct a hands-on technical POC: attempt multiple scraping methods against Yad2, document behaviors, research rate limits and legal restrictions, and produce a feasibility verdict with recommended approach.

## 3. Scope

### 3.1 In scope

**3.1.1 Scraping method testing**

Test each method against Yad2 rental listing pages and document results:
- Plain `requests.get` with various User-Agent headers
- Playwright headless with default configuration
- Playwright with stealth plugins (playwright-extra, stealth.min.js)
- Selenium with undetected-chromedriver
- Mobile User-Agent simulation (Yad2 mobile web)
- curl with cookie replay

For each method document: success/failure, Cloudflare challenge type encountered, data received (if any), detection signals observed.

**3.1.2 API endpoint research**

- Intercept Yad2 mobile app traffic (if possible) to identify REST/GraphQL endpoints
- Research known undocumented Yad2 API endpoints (community forums, GitHub, Stack Overflow)
- Check for RSS feeds, sitemap.xml, or other machine-readable data
- Test Yad2 search API parameters (city, category, price range)

**3.1.3 Rate limit and blocking research**

- Document request volume before block triggers (requests/minute, requests/hour)
- Document block duration and recovery behavior
- Test IP-based vs fingerprint-based blocking
- Research residential proxy effectiveness
- Document Cloudflare challenge progression (JS challenge → CAPTCHA → hard block)
- Research official Yad2 API program (if exists) and terms

**3.1.4 Data schema mapping**

For any successfully fetched listings, document:
- Available fields (title, price, location, rooms, area_sqm, floor, photos, contact info, description)
- Hebrew text handling (encoding, RTL markers, mixed Hebrew/English)
- Price format (NIS, monthly/annual, with/without utilities)
- Location format (city, neighborhood, street, coordinates)
- Field mapping to existing `ScrapedListing` dataclass (gaps and extensions needed)

**3.1.5 Legal and ToS assessment**

- Review Yad2 Terms of Service (scraping clauses)
- Research Israeli law on web scraping (Computer Law 5755-1995, Privacy Protection Law)
- Document known legal precedents (Yad2 vs scrapers, if any)
- Research official data partnership or API programs
- Risk assessment: likelihood and consequences of ToS violation

**3.1.6 Maintenance burden estimate**

- How often does Yad2 change its anti-bot measures? (community reports)
- What is the expected maintenance cost (hours/month) to keep a scraper working?
- Is the maintenance burden sustainable for our team size?

### 3.2 Out of scope (explicit)

- Production-quality scraper implementation
- Integration with existing shaked-wg-agent codebase
- Connector architecture design (that's S005-P001-WP002)
- Other Israeli platforms (Homeless, Facebook, etc.)
- Scoring model changes
