## 1. Scope Reminder

This WP replaces the hardcoded scraper mapping in `runner.py` (lines 113–127) with a data-driven registry that resolves scraper classes from fully-qualified names in `sources.json`. It introduces a `connector_class` field on `SourceDefinition` / `ResolvedSource` for the Yad2-style normalizer gateway pattern and extracts the flatfox-specific verification logic from the runner into the scraper module.

### 2.3 Dynamic Class Resolution — Replace Hardcoded Mapping

**File:** `shaked_wg_agent/runner.py` (lines 113–127)

**What to implement:**

1. Remove the hardcoded scraper mapping dict and the three inline `from ... import` statements in `_build_scraper()`.
2. Add a new private function `_resolve_class(fqn: str) -> type` that:
   - Splits `fqn` on the last `.` to get `module_path` and `class_name`.
   - Calls `importlib.import_module(module_path)` to load the module.
   - Uses `getattr(module, class_name)` to retrieve the class.
   - Validates `issubclass(cls, BaseScraper)`. If not, raises `TypeError`. Note: connectors (resolved via `connector_class`) MUST also subclass `BaseScraper` — they implement the same `fetch_listings()` interface but internally normalize data from a non-standard source. There is no separate `BaseConnector` class; the distinction is semantic (how they acquire data), not structural.
   - Returns the class.
3. Modify `_build_scraper(source, city)` to:
   - Determine the FQN: `fqn = source.connector_class or source.scraper_class`.
   - Call `_resolve_class(fqn)` to get the class.
   - Instantiate: `cls(source_id=source.source_id, search_url=source.search_url, city=city)`.
4. Add `import importlib` at the top of `runner.py`.

**Acceptance criteria:**

- AC-06: `_resolve_class("shaked_wg_agent.scrapers.flatfox.FlatfoxScraper")` returns the `FlatfoxScraper` class.
- AC-07: `_resolve_class("nonexistent.module.SomeClass")` raises `ImportError`.
- AC-08: `_resolve_class` with a class that does not subclass `BaseScraper` raises `TypeError`.
- AC-09: `_build_scraper(source, city)` where `source.connector_class` is set uses `connector_class` (not `scraper_class`).
- AC-10: `_build_scraper(source, city)` where `source.connector_class is None` uses `scraper_class`.
- AC-11: `grep -n "mapping\s*=" shaked_wg_agent/runner.py` returns zero hits (no hardcoded mapping dict).

## 5. Error Handling

| Error case | Where | Exception type | Message template |
|------------|-------|----------------|------------------|
| FQN has no `.` separator | `_resolve_class()` | `ValueError` | `"Invalid FQN '{fqn}': must contain module path and class name separated by '.'"` |
| Module not found | `_resolve_class()` | `ImportError` | `"Cannot import module '{module_path}' for scraper class '{fqn}'"` |
| Class not found in module | `_resolve_class()` | `AttributeError` | `"Module '{module_path}' has no class '{class_name}'"` |
| Class is not a BaseScraper subclass | `_resolve_class()` | `TypeError` | `"Class '{class_name}' is not a subclass of BaseScraper"` |
| `connector_class` set but unresolvable | `_build_scraper()` | `ImportError` | Same as module-not-found — the FQN path is identical |

## 6. Out of Scope (Explicit)

1. **Yad2 connector implementation** — NOT built. This WP adds the `connector_class` field and resolution mechanism. The actual Yad2 connector is S005-P001-WP003 scope.
2. **Generic post-scrape hook framework** — NOT built. Only flatfox has source-specific verification today. A generic hook interface is premature (YAGNI).
3. **FQN validation at JSON schema level** — NOT implemented. FQN is validated at runtime via `_resolve_class()`. No JSON schema enforcement.
4. **Caching resolved classes** — NOT built. Classes are resolved per-scan. Scan frequency (3x/day) makes caching overhead negligible and unnecessary.
5. **Config-time FQN validation** — NOT built at load time. FQN is validated when `_build_scraper` is called (at scan time). This avoids importing all scraper modules at config load.
