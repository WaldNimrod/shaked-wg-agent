## 1. Scope Reminder

Implement the **PostgreSQL schema**, **Alembic migrations**, **repository layer**, **JSON→PostgreSQL import tooling**, and **dual-backend persistence** controlled by environment variables. When `PERSISTENCE_BACKEND=postgresql`, the agent loads `ProjectConfig` from SQL (legacy tenant) and reads/writes listings and runs through tenant-scoped repositories. When `PERSISTENCE_BACKEND=json` (default), behavior matches pre-S003 code paths.

**In scope:** DDL matching §5 of LOD300; SQLAlchemy 2.x models; Alembic revision chain; `contextvars`-based active `tenant_id`; import CLI; parity tests for listing/run persistence APIs.

**Out of scope:** JWT, password storage, RLS policies, FastAPI route changes, billing — per LOD300 §1.4.

## 5. SQLAlchemy Models — Field Mapping (LOD300 §5)

Implement tables: `tenants`, `users` (+ nullable `external_subject TEXT`), `tenant_memberships`, `cities`, `sources`, `search_profiles`, `agent_settings`, `listings`, `runs`.

**Constraints (must match DB):**

1. `UNIQUE (tenants.slug)`  
2. `UNIQUE (users.email)`  
3. `PRIMARY KEY (tenant_memberships.tenant_id, tenant_memberships.user_id)`  
4. `UNIQUE (search_profiles.tenant_id, search_profiles.profile_slug)`  
5. `UNIQUE (listings.tenant_id, listings.listing_key)`  
6. `UNIQUE (runs.tenant_id, runs.run_key)`  
7. **FK:** `listings.profile_id` → `search_profiles(id)` ON DELETE SET NULL  
8. **FK:** `runs.profile_id` → `search_profiles(id)` ON DELETE CASCADE  
9. **Tenant alignment:** repositories **must** reject or skip inserts where `search_profiles.tenant_id != listings.tenant_id` (enforced in Python; DB does not express cross-table tenant equality on FK).

**Indexes:** per LOD300 — `listings (tenant_id, city_id)`, `listings (tenant_id, relevance_score DESC NULLS LAST)`, `runs (tenant_id, run_timestamp DESC)`.

**Acceptance criteria**

- [ ] AC-05: Alembic `upgrade` creates all relations; `alembic downgrade -1` drops cleanly on empty DB.
- [ ] AC-06: Repository unit test: inserting a listing row with `profile_id` pointing at a `search_profiles` row whose `tenant_id` differs from `listings.tenant_id` raises `ValueError` (or transaction rollback).

## 8. Config Loading — PostgreSQL Mode

When `PERSISTENCE_BACKEND=postgresql`:

1. Connect using `SHAKED_DATABASE_URL`.  
2. Resolve `tenant_id` = `SELECT id FROM tenants WHERE slug = :slug` with `slug=os.environ.get("SHAKED_LEGACY_TENANT_SLUG","legacy")`. Not found → raise `FileNotFoundError`-style message listing slug.  
3. `set_current_tenant_id(tenant_id)`.  
4. Load `agent_settings` for `tenant_id`.  
5. Resolve profile slug: CLI `profile_id` arg or `default_profile_slug` from `agent_settings`.  
6. Load `search_profiles` row `(tenant_id, profile_slug)` → `definition` JSONB → deserialize to existing `SearchProfile` via **shared helper** `_profile_from_dict` (extract from `_load_profile` body in `config.py` into private function used by both JSON and PG paths).  
7. Load `cities` row `city_id` from profile → `definition` JSONB → `_city_from_dict` (extract from `_load_city` logic).  
8. Load `sources` table → build `list[SourceDefinition]` from each row’s `definition` JSONB (same shape as `sources.json` element).  
9. Resolve `ResolvedSource[]` — **identical algorithm** to `load_config()` lines 357–400 (copy or call shared function).

**AgentMeta:** Build from `agent_settings` + profile default: `AgentMeta(default_profile_id=<slug from step 5>, ...)`.

**Acceptance criteria**

- [ ] AC-10: With populated DB (fixtures), `load_config("default")` under PG returns `ProjectConfig` equal field-for-field to JSON mode for the same seed data (golden test).  
- [ ] AC-11: Missing `search_profiles` row raises `FileNotFoundError` with profile slug in message.

## 9. Persistence Parity Layer

**Refactor** `shaked_wg_agent/persistence.py`:

- Keep public function signatures: `load_listings`, `save_listings`, `upsert_listing`, `mark_stale_listings`, `load_runs`, `save_runs`, `append_run`, `last_run`.
- Internally dispatch:

```python
def _backend() -> str:
    return os.environ.get("PERSISTENCE_BACKEND", "json").lower()

# if _backend() == "json": ... existing code ...
# elif _backend() == "postgresql": delegate to db.repositories.*
```

### 9.1 Listings (PostgreSQL)

- `load_listings()` → SELECT `payload` FROM `listings` WHERE `tenant_id = current_tenant_id()`; merge `listing_key` into each dict as `listing_id` if payload lacks it (payload must contain `listing_id` per import — verify in tests).
- `upsert_listing` → INSERT … ON CONFLICT (tenant_id, listing_key) DO UPDATE (preserve user fields `status`, `note`, `tags` exactly as current JSON implementation).
- `mark_stale_listings` → same age logic as `persistence.py` using `last_seen_at` from payload or columns.

### 9.2 Runs (PostgreSQL)

- `load_runs` → ORDER BY `run_timestamp` DESC; payload dict must include `run_id` key from `run_key`.
- `append_run` → prepend semantics = insert new row with latest `run_timestamp`.

**Acceptance criteria**

- [ ] AC-12: Run golden test: JSON backend vs PG backend same `upsert_listing` observable behavior on fixture listing.  
- [ ] AC-13: `mark_stale_listings` removes same logical rows as JSON implementation on synthetic clock injection (unit test with frozen time optional; if too heavy, document manual verification step in test plan).

## 10. Import CLI — `scripts/import_json_to_postgres.py`

**Invocation:**

```bash
PERSISTENCE_BACKEND=postgresql SHAKED_DATABASE_URL=... \
  python scripts/import_json_to_postgres.py \
    [--tenant-slug legacy] \
    [--bootstrap-email bootstrap@local.invalid] \
    [--data-dir data]
```

**Steps (transactional per phase):**

| Phase | Action |
|-------|--------|
| P1 | `INSERT` tenant `slug`, `display_name='Legacy import'`, `status='active'` |
| P2 | `INSERT` user email bootstrap; `INSERT` membership `role='admin'` |
| P3 | Upsert all `data/cities/*.json` into `cities` |
| P4 | Upsert `sources.json` rows into `sources` |
| P5 | For each `profiles/*.json`, insert `search_profiles` with `definition` = full file JSON |
| P6 | Insert `agent_settings` from `agent.json` |
| P7 | Import `listings.json` — map `profile_id` string → `search_profiles.id` UUID via `(tenant, slug)`; set `payload` = full object; denormalize `city_id`, scores, timestamps |
| P8 | Import `runs.json` — map `profile_id` → UUID; `run_key` = `run_id` |

**Idempotency:** Re-run uses ON CONFLICT upsert on natural keys; row counts reported at end.

**Acceptance criteria**

- [ ] AC-14: Second import on same data yields zero new duplicates (assert counts).  
- [ ] AC-15: Import completes on sample `data/` from repo under 60s on dev laptop (document timing as NFR).

## Appendix A — Initial DDL (Authoritative — Must Match Alembic Revision)

Single migration `0001_s003_p001_initial_schema`. Order: extensions → tenants/users → global ref → profiles → agent_settings → listings → runs.

```sql
CREATE EXTENSION IF NOT EXISTS pgcrypto;

CREATE TABLE tenants (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  slug TEXT NOT NULL UNIQUE,
  display_name TEXT NOT NULL,
  status TEXT NOT NULL CHECK (status IN ('active', 'suspended')),
  created_at TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE TABLE users (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  email TEXT NOT NULL UNIQUE,
  display_name TEXT,
  external_subject TEXT,
  created_at TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE TABLE tenant_memberships (
  tenant_id UUID NOT NULL REFERENCES tenants(id) ON DELETE CASCADE,
  user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
  role TEXT NOT NULL CHECK (role IN ('admin', 'agent', 'viewer')),
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  PRIMARY KEY (tenant_id, user_id)
);

CREATE TABLE cities (
  city_id TEXT PRIMARY KEY,
  definition JSONB NOT NULL,
  updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE TABLE sources (
  source_id TEXT PRIMARY KEY,
  definition JSONB NOT NULL,
  updated_at TIMESTAMPTZ NOT NULL DEFAULT now()
);

CREATE TABLE search_profiles (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  tenant_id UUID NOT NULL REFERENCES tenants(id) ON DELETE CASCADE,
  profile_slug TEXT NOT NULL,
  definition JSONB NOT NULL,
  city_id TEXT NOT NULL REFERENCES cities(city_id),
  created_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  updated_at TIMESTAMPTZ NOT NULL DEFAULT now(),
  UNIQUE (tenant_id, profile_slug)
);

CREATE TABLE agent_settings (
  tenant_id UUID PRIMARY KEY REFERENCES tenants(id) ON DELETE CASCADE,
  default_profile_slug TEXT NOT NULL,
  manual_triggers_only BOOLEAN NOT NULL DEFAULT true,
  project_window_days INT NOT NULL,
  project_start DATE,
  project_end DATE
);

CREATE TABLE listings (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  tenant_id UUID NOT NULL REFERENCES tenants(id) ON DELETE CASCADE,
  profile_id UUID REFERENCES search_profiles(id) ON DELETE SET NULL,
  listing_key TEXT NOT NULL,
  payload JSONB NOT NULL,
  city_id TEXT NOT NULL REFERENCES cities(city_id),
  relevance_score INT,
  first_seen_at TIMESTAMPTZ,
  last_seen_at TIMESTAMPTZ,
  UNIQUE (tenant_id, listing_key)
);

CREATE INDEX idx_listings_tenant_city ON listings(tenant_id, city_id);
CREATE INDEX idx_listings_tenant_score ON listings(tenant_id, relevance_score DESC NULLS LAST);

CREATE TABLE runs (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  tenant_id UUID NOT NULL REFERENCES tenants(id) ON DELETE CASCADE,
  profile_id UUID NOT NULL REFERENCES search_profiles(id) ON DELETE CASCADE,
  run_key TEXT NOT NULL,
  payload JSONB NOT NULL,
  run_timestamp TIMESTAMPTZ NOT NULL,
  UNIQUE (tenant_id, run_key)
);

CREATE INDEX idx_runs_tenant_time ON runs(tenant_id, run_timestamp DESC);
```

**FK summary:** Single-column references to `search_profiles(id)`; **tenant_id** on child rows is filtered in queries; repository validates tenant alignment on write (AC-06).
