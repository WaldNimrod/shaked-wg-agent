# Israel Apartment Search: Source Mapping & Entry Assessment — LOD200 Concept

**work_package_id:** S002-RND-WP001

## 1. Problem Statement

The system currently covers Swiss cities only (flatfox, wgzimmer, wg-gesucht). Expanding to Israel is a strategic goal, but the Israeli rental market has fundamentally different platforms, data formats, languages (Hebrew, RTL), currency (NIS), geographic structures (neighborhoods vs. zip codes), and user signals (no tram lines, different roommate demographics). Before committing development resources, we need a comprehensive market assessment to determine feasibility, effort, and viability.

## 2. Proposed Solution (concept only)

Conduct a structured market research phase covering: exhaustive source identification, hands-on data sampling, quality and feasibility scoring, competitor landscape mapping, and a detailed adaptation analysis. The output is a single comprehensive report that serves as the entry gate for the Israel expansion decision.

## 3. Scope

### 3.1 In scope

**3.1.1 Source mapping (broad as possible)**

Identify all relevant Israeli apartment/room listing platforms. For each source document:
- Platform name, URL, and type (marketplace, classifieds, social, aggregator)
- Listing volume estimate (order of magnitude)
- API availability (public REST, GraphQL, undocumented, none)
- Scraping feasibility (static HTML, JS-rendered, SPA)
- Data schema — fields available (title, price, location, rooms, photos, contact, description)
- Language (Hebrew, English, mixed, Russian)
- Geographic coverage (national, city-specific, neighborhood-level)
- Login/registration requirements
- Anti-bot measures (CAPTCHA, rate limiting, Cloudflare, fingerprinting)
- Known target platforms to evaluate: yad2.co.il, homeless.co.il, onmap.co.il, madlan.co.il, winwin.co.il, Facebook groups, Telegram groups, komo.co.il, airdna (short-term), Airbnb (sublets), university housing boards

**3.1.2 Data sampling**

For each accessible source, attempt to fetch 5-10 sample listings. Document:
- Raw data format (JSON, HTML structure, API response)
- Field-by-field mapping to our `ScrapedListing` dataclass
- Data quality assessment: completeness (% of fields populated), freshness (posting dates), accuracy
- Hebrew text handling requirements (RTL, mixed Hebrew/English, transliteration)

**3.1.3 Source quality report**

Per-source assessment table with:
- Estimated listing count (rooms/shared apartments specifically, not just all real estate)
- Data completeness score (1-5)
- Scraping difficulty rating: easy / medium / hard / blocked
- Playwright required? (yes/no)
- Login/CAPTCHA required? (yes/no/partial)
- Update frequency (real-time, daily, weekly)
- Geographic granularity (city, neighborhood, street level)
- Relevance to WG/shared apartment search specifically (vs. full apartments, commercial, etc.)

**3.1.4 Competitor and complementary services**

Identify existing Israeli apartment search aggregators, bots, or services. For each:
- Name, URL, business model
- What they do (aggregation, matching, alerts, community)
- Target audience (students, young professionals, families, all)
- Data sources they aggregate
- Pricing (free, freemium, paid)
- Strengths and weaknesses
- Gaps our system could fill
- Potential partnership or API integration opportunities

**3.1.5 Adaptation recommendations**

Detailed report on system changes required for Israel:
- **Scrapers:** New implementations needed per source, estimated complexity
- **Scoring model:** Israeli signals differ from Swiss — no tram lines (bus/light rail/train instead), dietary signals less relevant, army service age patterns (post-army roommates age 21-24), different price ranges (NIS), area preferences (proximity to university, beach, city center)
- **City definitions:** Israeli geography uses neighborhoods/שכונות more than zip codes; smaller cities may not have well-defined bbox
- **Language:** Hebrew RTL support in notifications, digest builder, HTML reports; mixed Hebrew/English listings; potential need for transliteration
- **Currency:** NIS (₪) instead of CHF; conversion for comparison
- **Notification localization:** Hebrew message templates, RTL email/Telegram formatting
- **Legal considerations:** Israeli privacy law, terms of service compliance per platform

**3.1.6 Entry assessment**

Go/no-go recommendation with:
- Recommended first Israeli city (likely Tel Aviv due to volume)
- Estimated effort for first city launch (person-days)
- Minimum viable source set (which 2-3 platforms to start with)
- Critical blockers (if any)
- Phased rollout plan suggestion

### 3.2 Out of scope (explicit)

- Actual scraper implementation for Israeli sources
- Israeli city definition file creation
- Production deployment for Israel
- S003 multi-tenant integration
- Hebrew UI/UX design
- Legal review (identified as needed, not performed)
