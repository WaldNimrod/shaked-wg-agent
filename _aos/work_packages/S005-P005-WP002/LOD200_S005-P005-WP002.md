**work_package_id:** S005-P005-WP002
**domain:** shaked-wg-agent
**track:** A — new scraper subclass following BaseScraper pattern, single team
**priority:** HIGH — primary new data source for Israeli market
**approved_by:** _draft_v1.1.0_
**approved_at:** _awaiting_L-GATE_S_round_2_

## 1. Problem Statement

In the Israeli rental market, a significant portion of rental listings are posted in local Facebook groups (e.g., "דירות להשכרה בפרדס חנה", "דירות בחדרה"). These listings are **free-text posts in Hebrew** with no structured data — unlike the HTML tables of homeless.co.il or the JSON API of flatfox.ch. Automated Facebook scraping is impractical for a personal project due to:

- Facebook ToS explicitly prohibits scraping; accounts get banned quickly
- Israeli Privacy Protection Law (Amendment 13, effective August 2025) restricts automated personal data collection
- Private groups require login + membership
- High maintenance burden (Facebook changes break scrapers frequently)

However, the **parsing challenge** (extracting structured rental data from free-text Hebrew posts) can be solved independently of the **acquisition method**. An LLM (Claude/GPT) excels at extracting price, rooms, location, availability, and other fields from messy Hebrew text — at negligible cost (~$0.05 per 1,000 posts).

## 2. Proposed Solution (concept only)

Create a **ManualFacebookScraper** — a BaseScraper subclass that accepts raw text input (copy-pasted from Facebook) rather than fetching from a URL. The scraper:

1. Reads raw post text from a local input file (JSON array of posts with text + metadata)
2. Sends each post to an LLM for structured extraction (price, rooms, city, neighborhood, availability, contact method, key features)
3. Filters: is this a rental offer? (vs. request, question, off-topic)
4. Maps extracted fields to `ScrapedListing` dataclass
5. Deduplicates against existing listings (by normalized address + price + rooms)

The acquisition is intentionally manual: the user browses Facebook groups as a regular member, copies interesting posts (or uses a simple bookmarklet/browser extension), and pastes them into an input file. This approach:
- Has zero legal/ToS risk (you're reading posts you have access to as a group member)
- Requires no proxies, login automation, or anti-detection
- Solves the hardest problem (Hebrew NLP) at near-zero cost
- Is extensible: the same parsing pipeline works for any future acquisition method (Apify, email, API)

## 3. Scope

### 3.1 In scope

**3.1.1 ManualFacebookScraper class**

- New file: `shaked_wg_agent/scrapers/facebook_manual.py`
- Subclasses `BaseScraper`, implements `fetch_listings() -> list[ScrapedListing]`
- Instead of fetching a URL, reads from a local JSON input file:
  ```json
  [
    {
      "post_id": "unique_id_or_hash",
      "group_name": "דירות להשכרה בפרדס חנה",
      "group_url": "https://www.facebook.com/groups/XXXXX",
      "text": "דירת 3 חדרים להשכרה בפרדס חנה, רחוב הרצל 15, 4500 ש\"ח...",
      "author_name": "",
      "posted_at": "2026-04-15T10:30:00",
      "has_images": true,
      "raw_url": "https://www.facebook.com/groups/XXXXX/posts/YYYYY"
    }
  ]
  ```
- Input file path configured via `search_url` field in sources.json (reuse existing field as file path for manual scrapers)
- If input file does not exist or is empty, returns `[]` (graceful)

**Normative input schema contract:**

| Field | Type | Required | Constraints | Default | Invalid behavior |
|-------|------|----------|-------------|---------|-----------------|
| `post_id` | `string` | YES | Non-empty, unique within file | — | Row skipped, warning logged |
| `group_name` | `string` | NO | — | `""` | Treated as empty string |
| `group_url` | `string` | NO | Informational; no runtime validation | `""` | Ignored (informational only) |
| `text` | `string` | YES | Non-empty, min 10 chars | — | Row skipped, warning logged |
| `author_name` | `string` | NO | — | `""` | Never propagated to output |
| `posted_at` | `string` | NO | ISO 8601 format if present | `null` | Parsed as `null`, warning logged |
| `has_images` | `boolean` | NO | — | `false` | Coerced to boolean |
| `raw_url` | `string` | NO | Informational; no runtime validation | `""` | Stored as-is in `direct_url` |

- File must be a valid JSON array. If JSON parsing fails: return `[]`, log error.
- Rows missing required fields (`post_id`, `text`) are skipped individually; valid rows in the same batch are still processed.
- Duplicate `post_id` within a single file: first occurrence wins, subsequent skipped with warning.

**3.1.2 LLM-based Hebrew post parsing**

- New file: `shaked_wg_agent/parsers/llm_listing_parser.py`
- Function: `parse_rental_post(text: str, group_name: str) -> dict | None`
- Uses an LLM API (Claude or OpenAI) to extract structured fields from free-text Hebrew
- Extraction prompt requests JSON output with fields:
  - `is_rental_offer` (bool) — filter out requests, questions, off-topic
  - `price_ils` (int | null)
  - `rooms` (float | null) — supports 3.5, 2, etc.
  - `city` (str)
  - `neighborhood` (str)
  - `street` (str)
  - `available_from` (str | null) — ISO date or null for immediate
  - `property_type` (str) — apartment, room, unit, villa, etc.
  - `floor` (int | null)
  - `area_sqm` (int | null)
  - `key_features` (list[str]) — parking, elevator, balcony, pets, etc.
  - `contact_method` (str) — "phone", "whatsapp", "dm", "comments"
- LLM provider configurable via environment variable (`SHAKED_LLM_PROVIDER=claude|openai`)
- API key via environment variable (`ANTHROPIC_API_KEY` or `OPENAI_API_KEY`)
- Cost: ~$0.05-0.10 per 1,000 posts (Haiku/GPT-4o-mini)

**LLM failure behavior matrix:**

| Failure mode | Scope | Behavior | Logging |
|-------------|-------|----------|---------|
| No API key configured (`ANTHROPIC_API_KEY` / `OPENAI_API_KEY` unset) | Run-level | `fetch_listings()` returns `[]` immediately, no LLM calls attempted | WARNING: "No LLM API key; facebook-manual scraper disabled" |
| No provider configured (`SHAKED_LLM_PROVIDER` unset) | Run-level | `fetch_listings()` returns `[]` immediately | WARNING: "No LLM provider; facebook-manual scraper disabled" |
| API timeout (single post) | Post-level | `parse_rental_post()` returns `None` for that post; remaining posts continue | WARNING per skipped post with post_id |
| API rate limit (429) | Post-level | Retry once with 2s backoff; if still 429, skip post, continue batch | WARNING per skipped post |
| Malformed LLM response (not valid JSON) | Post-level | `parse_rental_post()` returns `None`; post skipped, batch continues | WARNING with raw response snippet (first 200 chars) |
| API error (500, network) | Post-level | Skip post, continue batch | ERROR per skipped post |
| All posts in batch fail | Run-level | `fetch_listings()` returns `[]` (empty list, not error) | ERROR: "All {N} posts failed LLM parsing" |

The invariant: LLM failures never raise exceptions from `fetch_listings()`. The method always returns `list[ScrapedListing]`.

**3.1.3 Privacy-safe field handling**

- The parser MUST NOT store personal contact information (phone numbers, names) in ScrapedListing
- `author_name` from input is never propagated to output
- Phone numbers in post text are detected and stripped from `summary` field
- `contact_method` stores only the TYPE ("whatsapp", "phone") not the actual number
- This aligns with Israeli Privacy Protection Law requirements

**3.1.4 Deduplication logic**

- Before adding a listing, check against existing listings in `listings.json`:
  - Fuzzy match on: location_text + price (+/- 10%) (note: `ScrapedListing` base model has no `rooms` field; dedup uses the two dimensions available on the dataclass)
  - If match found with same source ("facebook-manual"), skip (duplicate within FB)
  - If match found with different source (e.g., "homeless"), mark as `cross_listed: true` (optional metadata)
- Within a single batch: hash on normalized text to catch exact cross-group duplicates

**3.1.5 Source registration in sources.json**

- Add new source entry:
  ```json
  {
    "source_id": "facebook-manual",
    "label": "Facebook Groups (manual)",
    "base_url": "https://www.facebook.com",
    "scraper_class": "shaked_wg_agent.scrapers.facebook_manual.ManualFacebookScraper",
    "requires_playwright": false,
    "notes": "Manual input from FB groups. LLM parsing. No automated scraping.",
    "city_params": {
      "pardes-hanna-region": {
        "search_url": "data/facebook/pardes-hanna-posts.json",
        "connection_method": "manual_file",
        "enabled": true
      }
    }
  }
  ```

**3.1.6 Update pardes-hanna profile**

- Add `"facebook-manual"` to `enabled_sources` in `data/profiles/pardes-hanna.json`

**3.1.7 CLI integration**

- `shaked-wg-agent run --profile pardes-hanna` picks up FB posts from the input file alongside homeless.co.il listings
- No special flags needed — the manual scraper reads from file just like other scrapers fetch from URLs

### 3.2 Out of scope (explicit)

- Automated Facebook scraping (no browser automation, no login, no Apify)
- Browser extension or bookmarklet for post capture (future enhancement)
- WhatsApp bot interface for post submission (future enhancement)
- Web UI for post input (future enhancement)
- Image/photo analysis from FB posts
- Cross-platform deduplication scoring (simple match only, no ML)
- Storing or processing personal contact information
- Facebook Graph API integration (deprecated for groups)

## 6. Risks

| Risk | Likelihood | Impact | Mitigation |
|------|-----------|--------|-----------|
| LLM API not available in Cowork sandbox | HIGH | MED | Provide mock/stub responses for testing; real LLM tested post-deployment |
| Hebrew parsing accuracy below useful threshold | LOW | MED | Claude/GPT excel at Hebrew; provide test cases in LOD400 |
| Users don't adopt manual copy-paste workflow | MED | MED | Low friction: simple JSON format; future phases add easier input methods |
| Cost creep from LLM API calls | LOW | LOW | Haiku/GPT-4o-mini at ~$0.05/1K posts; cap batch size |
| Privacy violation from stored personal data | MED | HIGH | Strip PII by design; never store phone/name in output |
| Deduplication false positives (different listings matched) | LOW | LOW | Conservative matching (require location_text + price(±10%) both match) |

## 7. Success Criteria (LOD200 level)

All criteria must be verifiable by automated test or deterministic inspection.

- [ ] SC-01: `ManualFacebookScraper` subclasses `BaseScraper` and implements `fetch_listings() -> list[ScrapedListing]` (verify: `issubclass(ManualFacebookScraper, BaseScraper)` assertion in test)

- [ ] SC-03: LLM parser returns valid structured output for a test corpus of ≥10 Hebrew rental posts; evaluation protocol: run `parse_rental_post()` on each post, assert output contains non-null `price_ils` OR `rooms` for ≥8/10 posts (80% threshold on fixed test set shipped with LOD400) (verify: pytest parametrized test with frozen test corpus in `tests/fixtures/hebrew_posts.json`)
- [ ] SC-04: For ≥3 non-rental posts in the test corpus (request, question, off-topic), `is_rental_offer` returns `false` (verify: pytest with labeled non-rental fixtures)
- [ ] SC-05: No `ScrapedListing` output contains phone numbers (regex `\d{9,10}`) or `author_name` field content (verify: unit test scans all output `.summary` and `.direct_url` fields)
