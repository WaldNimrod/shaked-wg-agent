# Pardes Hanna Region — Geographic Definition — LOD200 Concept

**work_package_id:** S005-P002-WP001
**domain:** shaked-wg-agent
**track:** A — direct to LOD400 (data file creation, pattern-following)
**approved_by:** _pending_
**approved_at:** _pending_

## 1. Problem Statement

The system's `CityDefinition` model was designed for Swiss cities with bounding boxes and zip codes. Israel expansion targets the Pardes Hanna region — not a single city but a geographic area encompassing multiple settlements. Israel does not use zip codes the same way as Switzerland. A new city definition file is needed that covers the region using bounding box coordinates and settlement-based filtering.

## 2. Proposed Solution (concept only)

Research the geographic boundaries of the Pardes Hanna region per Team 00's definition (north of Netanya, south of Haifa, east to Afula, excluding those 3 cities). Create a `data/cities/pardes-hanna-region.json` file with `country: "IL"`, a regional bounding box, and settlement-level filtering.

## 3. Scope

### 3.1 In scope

**3.1.1 Geographic research**

- Map all settlements/towns in the defined area:
  - Center: Pardes Hanna-Karkur
  - South: north of Netanya (Netanya excluded)
  - North: south of Haifa (Haifa excluded)
  - East: west of Afula (Afula excluded)
- Determine bounding box coordinates (west, east, south, north)
- List included settlements: Pardes Hanna-Karkur, Zikhron Ya'akov, Binyamina-Giv'at Ada, Caesarea, Or Akiva, Hadera, and others in the area
- Confirm excluded cities: Netanya, Haifa, Afula

**3.1.2 City definition file**

Create `data/cities/pardes-hanna-region.json`:
```json
{
  "city_id": "pardes-hanna-region",
  "city_name": "Pardes Hanna Region",
  "country": "IL",
  "bounding_box": { ... },
  "zip_filter": [],
  "available_sources": ["yad2"]
}
```

Notes:
- `zip_filter` likely empty — Israeli zip codes don't map well to this use case
- `available_sources` depends on Yad2 POC outcome (S005-P001-WP001)
- May need settlement name filter (new field) or coordinate-based filtering

**3.1.3 CityDefinition compatibility assessment**

- Does the existing `CityDefinition` dataclass support `country: "IL"`? (Yes — field exists)
- Does `zip_filter: []` work correctly? (Should — means no zip filtering)
- Are there any hardcoded "CH" assumptions in config.py or runner.py?

### 3.2 Out of scope (explicit)

- Scraper implementation for Israeli sources
- Scoring model changes
- Hebrew language support
- Profile creation for this region
