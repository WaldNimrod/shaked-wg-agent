## 1. Scope Reminder

This WP creates city definition files for Basel, Zurich, and Bern under the three-entity model introduced in WP001, populates the global source registry with per-city connection parameters, creates the default search profile targeting Basel with Shaked's preferences, and writes the agent metadata file. It also migrates the old flat config files to `.bak` backups. This is a data-creation WP — no application logic changes.

## 3. Data Model Changes

### 3.1 CityDefinition Schema

Each file in `data/cities/{city_id}.json` must conform to:

```json
{
  "$schema": "http://json-schema.org/draft-07/schema#",
  "type": "object",
  "required": ["city_id", "city_name", "country", "bounding_box", "zip_filter", "available_sources"],
  "properties": {
    "city_id": { "type": "string", "pattern": "^[a-z]+$" },
    "city_name": { "type": "string" },
    "country": { "type": "string", "enum": ["CH"] },
    "bounding_box": {
      "type": "object",
      "required": ["west", "east", "south", "north"],
      "properties": {
        "west": { "type": "number" },
        "east": { "type": "number" },
        "south": { "type": "number" },
        "north": { "type": "number" }
      },
      "additionalProperties": false
    },
    "zip_filter": {
      "type": "array",
      "items": { "type": "string", "pattern": "^[0-9]{4}$" }
    },
    "available_sources": {
      "type": "array",
      "items": { "type": "string" }
    }
  },
  "additionalProperties": false
}
```

**Validation rules:**
- `bounding_box`: `east > west`, `north > south`, span < 0.5 degrees per axis
- `zip_filter`: all entries must be 4-digit Swiss postal codes
- `available_sources`: each entry must match a `source_id` in `data/sources.json`

### 3.2 SourceDefinition Schema

`data/sources.json` is an array where each element conforms to:

```json
{
  "$schema": "http://json-schema.org/draft-07/schema#",
  "type": "object",
  "required": ["source_id", "label", "base_url", "scraper_class", "requires_playwright", "notes", "city_params"],
  "properties": {
    "source_id": { "type": "string" },
    "label": { "type": "string" },
    "base_url": { "type": "string", "format": "uri" },
    "scraper_class": { "type": "string" },
    "requires_playwright": { "type": "boolean" },
    "notes": { "type": "string" },
    "city_params": {
      "type": "object",
      "additionalProperties": {
        "type": "object",
        "required": ["search_url", "connection_method", "enabled"],
        "properties": {
          "search_url": { "type": "string" },
          "connection_method": { "type": "string", "enum": ["canton", "bbox", "city_id_param"] },
          "enabled": { "type": "boolean" }
        },
        "additionalProperties": false
      }
    }
  },
  "additionalProperties": false
}
```

## 5. Error Handling Requirements

| Error case | Expected behavior |
|-----------|-------------------|
| `data/cities/` directory does not exist | Create it before writing city files |
| `data/profiles/` directory does not exist | Create it before writing profile files |
| Old `data/config.json` does not exist | Skip `.bak` rename, proceed normally |
| Old `data/sources.json` does not exist | Skip `.bak` rename, proceed normally |
| Old `data/config.json.bak` already exists | Overwrite existing `.bak` (idempotent) |
| Old `data/sources.json.bak` already exists | Overwrite existing `.bak` (idempotent) |
| City file already exists at target path | Overwrite with the content specified in section 2.1 (idempotent creation) |
| `city_id` in filename does not match `city_id` field in JSON | Validation error -- reject at config load time (WP001 responsibility) |
| `enabled_sources` in profile references a source not in city `available_sources` | Validation warning at config load time (WP001 responsibility) |
| flatfox bbox params in sources.json do not match city bounding_box | Builder must verify manually -- values are copied from CityDefinition |

## 6. Out of Scope (Explicit)

- **Scraper code changes** -- scrapers already accept city parameters via WP001. No scraper modifications in this WP.
- **Search profiles for Zurich or Bern** -- city definitions only. Profiles for these cities come in S003 (multi-user).
- **Notification channel configuration** -- `channels: []` is intentionally empty. Channel setup is a future WP.
- **config.py / load_config() changes** -- the three-entity loader is built in WP001. This WP only creates the data files it consumes.
- **Deletion of .bak files** -- old backups are kept for safety. Cleanup is a later WP.
- **wg-gesucht enablement** -- listed in source registry but disabled (requires auth/CAPTCHA). Enablement is a separate WP.
- **tutti expansion beyond Basel** -- only Basel has a tutti city_params entry. Other cities are not mapped.
