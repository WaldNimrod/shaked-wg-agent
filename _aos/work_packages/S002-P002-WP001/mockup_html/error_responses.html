Fehlermeldungen

Alle Fehler-Responses folgen einem einheitlichen Envelope-Format. Nachfolgend sind alle möglichen Fehlercodes mit Beispielen aufgeführt.

Übersicht Fehlercodes

400 BAD_REQUEST — Ungültige Abfrageparameter

Wird zurückgegeben, wenn Query-Parameter ungültige Werte oder Typen enthalten.

{
  "error": {
    "code": "BAD_REQUEST",
    "message": "Ungültige Abfrageparameter",
    "details": {
      "limit": "Wert muss zwischen 1 und 200 liegen"
    }
  },
  "meta": {
    "timestamp": "2026-04-12T14:45:00+00:00",
    "request_id": "req-err-400a"
  }
}
401 UNAUTHORIZED — Fehlender oder ungültiger API-Schlüssel

Wird zurückgegeben, wenn der X-API-Key Header fehlt oder einen ungültigen Wert enthält.

{
  "error": {
    "code": "UNAUTHORIZED",
    "message": "Fehlender oder ungültiger API-Schlüssel",
    "details": null
  },
  "meta": {
    "timestamp": "2026-04-12T14:45:01+00:00",
    "request_id": "req-err-401a"
  }
}
404 NOT_FOUND — Ressource nicht gefunden

Wird zurückgegeben, wenn die angeforderte Ressource (Listing, Run) nicht existiert.

{
  "error": {
    "code": "NOT_FOUND",
    "message": "Ressource nicht gefunden",
    "details": {
      "resource": "listing",
      "id": "flatfox-99999999"
    }
  },
  "meta": {
    "timestamp": "2026-04-12T14:45:02+00:00",
    "request_id": "req-err-404a"
  }
}
422 VALIDATION_ERROR — Validierungsfehler im Request-Body

Wird zurückgegeben, wenn der JSON-Body fehlende oder ungültige Felder enthält.

{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Validierungsfehler im Request-Body",
    "details": {
      "city_id": "Pflichtfeld fehlt"
    }
  },
  "meta": {
    "timestamp": "2026-04-12T14:45:03+00:00",
    "request_id": "req-err-422a"
  }
}
500 INTERNAL_ERROR — Interner Serverfehler

Wird zurückgegeben, wenn ein unerwarteter Fehler auf dem Server auftritt. Bitte den Vorfall mit der request_id melden.

{
  "error": {
    "code": "INTERNAL_ERROR",
    "message": "Interner Serverfehler",
    "details": null
  },
  "meta": {
    "timestamp": "2026-04-12T14:45:04+00:00",
    "request_id": "req-err-500a"
  }
}
Allgemeines Fehler-Envelope-Format

Jede Fehler-Response folgt diesem Schema:

{
  "error": {
    "code": "string    — Maschinenlesbarer Fehlercode",
    "message": "string — Menschenlesbare Fehlerbeschreibung",
    "details": "object | null — Zusätzliche Fehlerdetails"
  },
  "meta": {
    "timestamp": "string — ISO 8601 Zeitstempel",
    "request_id": "string — Eindeutige Request-ID"
  }
}
