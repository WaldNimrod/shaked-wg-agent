# Facebook Group Agent — Feasibility & Cost Evaluation — LOD200 Concept

## 1. Problem Statement

Facebook Groups are a major source for Israeli rental listings, especially roommate/WG posts (highest velocity for shared apartment listings). However, Facebook actively blocks automated scraping, enforces strict ToS against it, and provides limited API access for group content. Team 00 wants to evaluate whether running a browser agent for Facebook group monitoring is feasible and at what cost.

## 2. Proposed Solution (concept only)

Conduct a structured evaluation of browser agent approaches for Facebook group monitoring. Produce a cost analysis and go/no-go recommendation.

## 3. Scope

### 3.1 In scope

**3.1.1 Browser agent feasibility**

- Evaluate headless browser automation (Playwright, Puppeteer) for FB group scraping
- Test anti-detection measures: stealth plugins, residential proxies, browser fingerprint rotation
- Document Facebook's detection mechanisms (behavioral analysis, CAPTCHA, account restrictions)
- Assess account requirements: does scraping require a logged-in account? Risk of account ban?

**3.1.2 Cost analysis**

- Compute costs: server/cloud resources for running Playwright continuously
- Proxy costs: residential proxy pricing for Facebook (per GB or per request)
- Anti-detection tools: commercial solutions (Browserless, ScrapingBee, etc.) with FB support
- Account cost: if accounts get banned, cost of replacement
- Total monthly cost estimate for monitoring N groups at M frequency

**3.1.3 Alternative approaches**

- Meta Graph API: what access is available for group content? Application requirements?
- Manual curation: human monitors groups, adds listings manually
- Community partnerships: collaborate with group admins for data sharing
- RSS/notification bridges: any third-party tools that bridge FB groups to machine-readable formats?
- Telegram mirrors: some FB groups have Telegram mirrors — evaluate coverage

**3.1.4 Legal and ToS assessment**

- Facebook ToS on automated access
- Meta's enforcement history (lawsuits, cease-and-desist)
- Israeli law on social media scraping
- Data privacy considerations (personal posts, contact info)
- Risk classification: LOW / MEDIUM / HIGH / UNACCEPTABLE

### 3.2 Out of scope (explicit)

- Actual implementation of Facebook scraping
- Integration with shaked-wg-agent
- Other social platforms (Telegram — may be separate WP)
