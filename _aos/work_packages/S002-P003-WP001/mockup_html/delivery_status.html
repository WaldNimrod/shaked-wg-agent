Delivery Status im Run Record

Das notification_sent Feld im Run Record dokumentiert den Versandstatus nach jedem Scan-Lauf.

Erfolg
Beide Kanaele erfolgreich 
{
  "run_id": "run-20260412-143000-a1b2",
  "city_id": "basel",
  "new_results": 3,
  "notification_sent": {
    "email": true,
    "telegram": true,
    "errors": []
  }
}
Email gesendet Telegram gesendet
Teilweiser Fehler
Telegram fehlgeschlagen 
{
  "run_id": "run-20260412-143000-a1b2",
  "city_id": "basel",
  "new_results": 3,
  "notification_sent": {
    "email": true,
    "telegram": false,
    "errors": [
      "Telegram: HTTP 429 rate limited (retry exhausted)"
    ]
  }
}
Email gesendet Telegram fehlgeschlagen
Uebersprungen
Keine neuen Ergebnisse 
{
  "run_id": "run-20260412-143000-a1b2",
  "city_id": "basel",
  "new_results": 0,
  "notification_sent": null
}
Kein Versand

Wenn new_results gleich 0 ist, wird keine Benachrichtigung versendet und das Feld ist null.

notification_sent Feld-Referenz
Feld Typ Beschreibung
notification_sent object | null null wenn keine neuen Ergebnisse, sonst Objekt mit Versanddetails
.email bool true wenn Email erfolgreich versendet
.telegram bool true wenn Telegram-Nachricht erfolgreich versendet
.errors string[] Liste der Fehlermeldungen (leer bei Erfolg)
