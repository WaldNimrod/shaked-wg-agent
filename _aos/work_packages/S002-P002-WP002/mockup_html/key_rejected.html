Rejected Requests

Drei Ablehnungsszenarien fuer ungueltige oder fehlende Authentifizierung.

1. Fehlender Header
Request
GET /listings?city_id=basel HTTP/1.1
Host: api.shaked-wg.example.com

Kein X-API-Key Header gesendet.

Response
401 Unauthorized 
{
  "detail": "Invalid or missing API key"
}
2. Falscher Key
Request
GET /listings?city_id=basel HTTP/1.1
Host: api.shaked-wg.example.com
X-API-Key: wrong-key-value
Response
401 Unauthorized 
{
  "detail": "Invalid or missing API key"
}
3. Server-Fehlkonfiguration
Situation

API_KEY Umgebungsvariable ist nicht gesetzt. Der Server startet nicht oder gibt einen internen Fehler zurueck.

Response
500 Internal Server Error 
{
  "detail": "Server configuration error: API_KEY not set"
}
Sicherheitshinweis: Die Fehlermeldung ist fuer fehlenden und falschen Schluessel identisch (kein Information Leakage). Ein Angreifer kann nicht unterscheiden, ob der Header fehlt oder der Schluessel falsch ist.
