## 1. Scope Reminder

Create `data/cities/pardes-hanna-region.json` — the first Israeli city definition. Follows the `basel.json` pattern exactly. `country: "IL"` and `currency: "ILS"` enable the internationalization infrastructure from S005-P004. `zip_filter` is empty (Israeli zip codes don't map to regions like Swiss PLZ). `available_sources` lists the Israeli sources defined in WP002.

---

## 2. Technical Specification

### 2.1 City Definition File — `data/cities/pardes-hanna-region.json`

**What to implement:**

Create the file with this exact content:

```json
{
  "city_id": "pardes-hanna-region",
  "city_name": "Pardes Hanna Region",
  "country": "IL",
  "currency": "ILS",
  "bounding_box": {
    "west": 34.87,
    "east": 35.00,
    "south": 32.42,
    "north": 32.60
  },
  "zip_filter": [],
  "available_sources": ["homeless"]
}
```

**Geographic coverage:**

| Settlement | Latitude | Longitude | Status |
|-----------|----------|-----------|--------|
| Pardes Hanna-Karkur | 32.4728 | 34.9742 | INCLUDED (center) |
| Zikhron Ya'akov | 32.5739 | 34.9520 | INCLUDED |
| Binyamina-Giv'at Ada | 32.5182 | 34.9540 | INCLUDED |
| Caesarea | 32.5190 | 34.9045 | INCLUDED |
| Or Akiva | 32.5100 | 34.9204 | INCLUDED |
| Hadera | 32.4419 | 34.9039 | INCLUDED |
| Netanya | 32.33 | 34.86 | EXCLUDED (south of box) |
| Haifa | 32.82 | 34.99 | EXCLUDED (north of box) |
| Afula | 32.61 | 35.29 | EXCLUDED (east of box) |

**Acceptance criteria:**

- AC-01: File `data/cities/pardes-hanna-region.json` exists and is valid JSON.
- AC-02: `city_id` is `"pardes-hanna-region"` (lowercase, hyphenated).
- AC-03: `country` is `"IL"`.
- AC-04: `currency` is `"ILS"`.
- AC-05: Bounding box covers all 6 included settlements (west=34.87, east=35.00, south=32.42, north=32.60).
- AC-06: Bounding box excludes Netanya (32.33°N < 32.42°N south boundary).
- AC-07: Bounding box excludes Haifa (32.82°N > 32.60°N north boundary).
- AC-08: Bounding box excludes Afula (35.29°E > 35.00°E east boundary).
- AC-09: `zip_filter` is an empty array `[]`.
- AC-10: `available_sources` is `["homeless"]`.

### 2.2 Config Loading Verification

**File:** `shaked_wg_agent/config.py` — READ ONLY (no changes)

**What to verify:**

The existing `_load_city()` function must correctly load the new city file. Key checks:

1. `_load_city("pardes-hanna-region")` returns a `CityDefinition` with `country="IL"`, `currency="ILS"`.
2. Empty `zip_filter` does not cause errors downstream.
3. `available_sources=["homeless"]` is accepted (even if the "homeless" source is not yet in `sources.json` — config loading does not validate source existence).

**Acceptance criteria:**

- AC-11: `python3 -c "from shaked_wg_agent.config import _load_city; c = _load_city('pardes-hanna-region'); print(c.country, c.currency)"` prints `IL ILS`.
- AC-12: `_load_city('pardes-hanna-region').bounding_box.west == 34.87` (spot check).

## 3. Data Model

No changes to `CityDefinition` dataclass. The new file uses existing fields only.

| Field | Type | Value | Notes |
|-------|------|-------|-------|
| city_id | str | "pardes-hanna-region" | Matches filename |
| city_name | str | "Pardes Hanna Region" | Display name |
| country | str | "IL" | Triggers LOCALE_IL |
| currency | str | "ILS" | ISO 4217 for Israeli New Shekel |
| bounding_box | BoundingBox | {west: 34.87, east: 35.00, south: 32.42, north: 32.60} | Regional box |
| zip_filter | list[str] | [] | Not used for Israeli geography |
| available_sources | list[str] | ["homeless"] | Updated when more IL sources added |
