═══ PROJECT ═══

shaked-wg-agent is a Python CLI application that scrapes rental listings from multiple platforms, scores them by relevance for a specific user profile, and delivers results via CLI, HTML report, REST API, and notifications. The codebase was recently internationalized (S005-P004) — all fields are currency-agnostic, locale-aware, and support country="IL".

Key architecture:
  data/cities/{city_id}.json → config.py (CityDefinition) → runner.py → scrapers
  data/profiles/{profile_id}.json → config.py (SearchProfile) → scorer.py
  data/sources.json → config.py (SourceDefinition) → runner.py → _resolve_class() via importlib
  shaked_wg_agent/locale.py → LOCALE_IL already defined (Hebrew keywords, ₪, RTL, Accept-Language)

ScrapedListing fields: source, source_listing_id, title, price (int|None), currency (str), country (str), available_from, location_text, district, transit_match_lines, roommate_signal, vegan_signal, summary, direct_url, url_status, posted_date.

═══ SCRAPER PATTERN (reference: flatfox.py) ═══

A scraper subclasses BaseScraper and implements fetch_listings() → list[ScrapedListing]:
  1. __init__(self, source_id, search_url, city) — called by runner
  2. fetch_listings() — HTTP or Playwright requests, parse HTML/JSON, return ScrapedListing list
  3. Each listing sets: price=..., currency=city.currency, country=city.country
  4. Use self._get(url) for plain HTTP, or Playwright for sites with anti-bot (see wgzimmer_pw.py)
  5. Use self._soup(html) for BeautifulSoup parsing
