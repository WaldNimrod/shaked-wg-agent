## 1. Scope Reminder

Replace the standard Playwright dependency with Patchright (drop-in fork removing automation markers) and switch from ephemeral browser contexts to a persistent browser profile that retains Google cookies across sessions. This restores wgzimmer.ch search results by improving the reCAPTCHA v3 score from 0.1-0.3 (blocked) to 0.7-0.9 (passed). No parsing/filtering logic is changed.

### 2.2 Switch to Persistent Browser Context

**File:** `shaked_wg_agent/scrapers/wgzimmer_pw.py`

**What to implement:**

1. Add `os` and `pathlib.Path` imports at the top of the file (if not already present).
2. Define the profile directory constant after imports:
   ```python
   _PROFILE_DIR = os.environ.get(
       "SHAKED_BROWSER_PROFILE_DIR",
       os.path.expanduser("~/.shaked-wg/browser-profile/wgzimmer")
   )
   ```
3. Replace the current browser launch + context creation block (currently around lines 75-85):
   ```python
   # BEFORE:
   browser = pw.chromium.launch(headless=True)
   ctx = browser.new_context(
       user_agent="Mozilla/5.0 ...",
       locale="de-CH",
   )
   page = ctx.new_page()
   ```
   with:
   ```python
   # AFTER:
   os.makedirs(_PROFILE_DIR, exist_ok=True)
   ctx = pw.chromium.launch_persistent_context(
       user_data_dir=_PROFILE_DIR,
       headless=True,
       user_agent="Mozilla/5.0 (Windows NT 10.0; Win64; x64) AppleWebKit/537.36 (KHTML, like Gecko) Chrome/120.0.0.0 Safari/537.36",
       locale="de-CH",
       viewport={"width": 1920, "height": 1080},
   )
   page = ctx.pages[0] if ctx.pages else ctx.new_page()
   ```
4. Remove `browser.close()` in the finally block — persistent context uses `ctx.close()` instead. The `browser` variable no longer exists; the persistent context IS the browser.
5. Update the `try/finally` structure:
   ```python
   # BEFORE:
   browser = pw.chromium.launch(headless=True)
   try:
       ctx = browser.new_context(...)
       # ... work ...
   finally:
       browser.close()

   # AFTER:
   os.makedirs(_PROFILE_DIR, exist_ok=True)
   ctx = pw.chromium.launch_persistent_context(...)
   try:
       page = ctx.pages[0] if ctx.pages else ctx.new_page()
       # ... work ...
   finally:
       ctx.close()
   ```

## 5. Error Handling

**Exception boundary contract:** The entire `fetch_listings()` method body (after the import guard) MUST be wrapped in a single top-level `try/except Exception` block. Any exception from Patchright, the persistent context, or page operations results in `return []` with an ERROR-level log. This is the explicit catch boundary.

```python
def fetch_listings(self) -> list[ScrapedListing]:
    if not _HAS_PATCHRIGHT:
        logger.warning("patchright not installed — wgzimmer scraper disabled")
        return []
    
    try:
        os.makedirs(_PROFILE_DIR, exist_ok=True)
        with sync_playwright() as pw:
            ctx = pw.chromium.launch_persistent_context(...)
            try:
                # ... page operations, API interception, parsing ...
                pass
            finally:
                ctx.close()
    except Exception as e:
        logger.error("wgzimmer scraper failed: %s", e)
        return []
    
    return results
```

| Error Case | Where | Exception Type | Catch Boundary | Behavior | Log Level |
|------------|-------|---------------|----------------|----------|-----------|
| `patchright` not installed | Module import | `ImportError` | Import guard (`_HAS_PATCHRIGHT`) | Return `[]` | WARNING |
| Profile directory creation fails (permissions) | `os.makedirs()` | `OSError` | Top-level `try/except` | Return `[]` | ERROR |
| Profile directory locked by concurrent scan | `launch_persistent_context()` | `playwright.Error` / `OSError` | Top-level `try/except` | Return `[]` | ERROR |
| Corrupt browser profile (invalid SQLite, etc.) | `launch_persistent_context()` | `playwright.Error` | Top-level `try/except` | Return `[]` | ERROR |
| reCAPTCHA blocks after Patchright | Content check | None (handled in flow) | Explicit `if` check | Return `[]` | WARNING |
| Page navigation timeout | `page.goto()` | `playwright.TimeoutError` | Top-level `try/except` | Return `[]` | ERROR |
| API interception fails | `_on_response` callback | `Exception` | Inner callback `try/except` (existing) | Silently continue | — |
| DOM fallback parsing fails | `_dom_fallback()` | `Exception` | Top-level `try/except` | Return `[]` | ERROR |

**Invariant:** `fetch_listings()` NEVER raises an exception. It always returns `list[ScrapedListing]`.
