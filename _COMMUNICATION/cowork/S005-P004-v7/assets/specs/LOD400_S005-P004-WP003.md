## 1. Scope Reminder

This WP extracts hardcoded German keywords and labels from scorer, scrapers, CLI, and notifiers into a locale-keyed registry. The registry is a Python module (`shaked_wg_agent/locale.py`) containing a frozen `Locale` dataclass with per-market keyword sets, labels, and language metadata. Two locales are defined: de-CH (Swiss German) and he-IL (Hebrew). A `get_locale(country)` accessor resolves locale from the city's country code. This WP provides the data layer for locale resolution; RTL rendering is WP004 scope.

## 2. Technical Specification

### 2.1 New Module: `shaked_wg_agent/locale.py`

**File:** `shaked_wg_agent/locale.py` (NEW)

**What to implement:**

1. Create a new module `shaked_wg_agent/locale.py`.
2. Define a frozen dataclass `Locale` with all fields specified in §3.1.
3. Define module-level constant `LOCALE_CH` with Swiss German locale data:
   - `accept_language = "de-CH,de;q=0.9,en;q=0.8"`
   - `vegan_strong = frozenset({"vegan", "vegane", "veganer", "veganes", "tierfreie küche", "tierfreie wg"})`
   - `vegan_partial = frozenset({"pflanzlich", "pflanzliche", "vegan-freundlich", "vegetarisch"})`
   - `vegan_weak = frozenset({"kein fleisch", "kein fisch", "plant-based"})`
   - `vegan_no_signal = frozenset({"kein signal", "unbekannt"})`
   - `status_labels = {"favorit": "⭐ favorit", "interessant": "interessant", "kontaktiert": "kontaktiert", "neu": "neu", "abgesagt": "abgesagt"}`
   - `transit_label = "Transit lines"`
   - `currency_symbol = "CHF"`
   - `direction = "ltr"`
   - `html_lang = "de"`
4. Define module-level constant `LOCALE_IL` with Hebrew locale data:
   - `accept_language = "he-IL,he;q=0.9,en;q=0.8"`
   - `vegan_strong = frozenset({"טבעוני", "טבעונית", "טבעוניים", "מטבח טבעוני"})`
   - `vegan_partial = frozenset({"צמחוני", "צמחונית", "ללא בשר", "plant-based"})`
   - `vegan_weak = frozenset({"ידידותי לטבעונים", "אפשרות טבעונית"})`
   - `vegan_no_signal = frozenset({"לא צוין", "לא ידוע"})`
   - `status_labels = {"favorit": "⭐ מועדף", "interessant": "מעניין", "kontaktiert": "נוצר קשר", "neu": "חדש", "abgesagt": "בוטל"}`
   - `transit_label = "Transit lines"`
   - `currency_symbol = "₪"`
   - `direction = "rtl"`
   - `html_lang = "he"`
5. Define `LOCALES: dict[str, Locale] = {"CH": LOCALE_CH, "IL": LOCALE_IL}`.
6. Define `_DEFAULT_LOCALE = LOCALE_CH`.
7. Define function `get_locale(country: str = "CH") -> Locale` that returns `LOCALES.get(country, _DEFAULT_LOCALE)`.

**Acceptance criteria:**

- AC-01: `Locale` is a frozen dataclass (`@dataclass(frozen=True)`) with 10 fields as specified in §3.1.
- AC-02: `get_locale("CH")` returns a `Locale` with `accept_language == "de-CH,de;q=0.9,en;q=0.8"`.
- AC-03: `get_locale("IL")` returns a `Locale` with `accept_language == "he-IL,he;q=0.9,en;q=0.8"`.
- AC-04: `get_locale("XX")` returns the Swiss German locale (fallback to `_DEFAULT_LOCALE`).
- AC-05: `get_locale("CH").vegan_strong` is a `frozenset` containing `"vegan"`, `"vegane"`, `"veganer"`, `"veganes"`, `"tierfreie küche"`, `"tierfreie wg"`.
- AC-06: `get_locale("IL").vegan_strong` is a `frozenset` containing `"טבעוני"`, `"טבעונית"`, `"טבעוניים"`, `"מטבח טבעוני"`.
- AC-07: `get_locale("CH").direction == "ltr"` and `get_locale("IL").direction == "rtl"`.
- AC-08: `get_locale("CH").html_lang == "de"` and `get_locale("IL").html_lang == "he"`.
- AC-09: `get_locale("CH").status_labels["neu"] == "neu"` and `get_locale("IL").status_labels["neu"] == "חדש"`.
- AC-10: `get_locale("CH").currency_symbol == "CHF"` and `get_locale("IL").currency_symbol == "₪"`.

**Cross-WP note on currency:** `Locale.currency_symbol` is the compact display symbol (`"CHF"`, `"₪"`), while `CityDefinition.currency` (WP001) is the ISO 4217 code (`"CHF"`, `"ILS"`). The mapping is: CH → ISO `"CHF"` / display `"CHF"`, IL → ISO `"ILS"` / display `"₪"`. Display consumers use `Locale.currency_symbol` for UI; data consumers use `CityDefinition.currency` for machine-readable ISO codes.

### 2.6 Email Notifier — Locale-Aware Text

**File:** `shaked_wg_agent/notifier/email_notifier.py` (lines 35, 41, 52, 71)
**File:** `shaked_wg_agent/locale.py` (extend — add module-level dicts, NOT Locale fields)

**What to implement:**

1. In `locale.py`, add module-level dicts for email text strings (separate from the `Locale` dataclass — `Locale` remains at exactly 10 fields as defined in §2.1/AC-01):
   ```
   EMAIL_STRINGS: dict[str, dict[str, str]]
   ```
   Keyed by country code (`"CH"`, `"IL"`), each containing:
   - `"new_offers"`: `"neue Angebote"` / `"הצעות חדשות"`
   - `"price_not_specified"`: `"Preis nicht angegeben"` / `"מחיר לא צוין"`
   - `"generated_by"`: `"Generiert von Shaked WG Agent"` / `"נוצר על ידי Shaked WG Agent"`
   Add accessor: `get_email_strings(country: str = "CH") -> dict[str, str]` with CH fallback.
2. In `email_notifier.py`, add import: `from shaked_wg_agent.locale import get_email_strings`.
3. Replace hardcoded German strings with lookups from `get_email_strings(country)`:
   - `"neue Angebote"` → `strings["new_offers"]`
   - `"Preis nicht angegeben"` → `strings["price_not_specified"]`
   - `"Generiert von Shaked WG Agent"` → `strings["generated_by"]`
4. The country code is derived from the digest payload (`digest_payload.get("country", "CH")`) or from the city associated with the profile.

**Design rationale:** Email strings are NOT added to the `Locale` dataclass because `Locale` is a frozen dataclass focused on scoring/display concerns (keywords, labels, direction). Email text is a separate concern and may grow independently. Module-level dicts in `locale.py` keep all locale data colocated without inflating the `Locale` contract.

**Acceptance criteria:**

- AC-26: `grep -n '"neue Angebote\|Preis nicht angegeben\|Generiert von"' shaked_wg_agent/notifier/email_notifier.py` returns zero hits (no hardcoded German text).
- AC-27: `locale.get_email_strings("CH")["new_offers"] == "neue Angebote"`.
- AC-28: `locale.get_email_strings("IL")["new_offers"] == "הצעות חדשות"`.
- AC-29: `Locale` dataclass remains at exactly 10 fields (email strings are NOT Locale fields).

## 5. Error Handling

| Error case | Where | Expected behavior |
|------------|-------|-------------------|
| Unknown country code in `get_locale()` | `locale.py` | Returns `_DEFAULT_LOCALE` (Swiss German) — no error raised |
| `Locale` with empty `vegan_strong` frozenset | `_vegan_score()` | Valid — `any(kw in lower for kw in frozenset())` returns `False`, score = 0 |
| Listing dict missing `"country"` key | `score_listing()` | `listing.get("country", "CH")` defaults to `"CH"` — no error |
| Status key not in `locale.status_labels` | CLI | `locale.status_labels.get(status, status)` — falls back to raw key |

## 6. Out of Scope (Explicit)

1. **RTL rendering** — NOT in this WP. `Locale.direction` and `Locale.html_lang` are data fields consumed by WP004 (HTML report i18n). This WP does not add `dir="rtl"`, `bootstrap.rtl.min.css`, or CSS logical properties. RTL implementation MUST follow AOS Module 11 (RTL_BIDI_STANDARD_v1.0.0.md) when implemented in WP004.
2. **HTML report string localization** — NOT in this WP. The ~50+ German strings in `html_report.py` are WP004 scope.
3. **Scraper parsing keyword localization** — NOT in this WP. German scrapers (wgzimmer, flatfox, wg-gesucht) parse German-language sites. Their parsing patterns are source-specific, not locale-specific. Israeli scrapers (future) will parse Hebrew natively.
4. **Additional locales beyond CH and IL** — NOT added. The registry supports it structurally, but only two locales are defined.
5. **Locale persistence to JSON** — NOT built. Locale is code-level (Python module), not config-level. Keywords change only with code releases.
6. **ntfy/telegram/discord notifier text localization** — NOT in this WP. Only email notifier text is parameterized (it has the most German text). Other notifiers pass through listing data which is already locale-aware via status labels and currency.
